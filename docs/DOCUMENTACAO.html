1. Visão Geral

Sistema SaaS de agendamento de serviços com dois painéis distintos:

  • Painel Administrativo — para admins e funcionários gerenciarem agenda, serviços, usuários e disponibilidade
  • Portal do Cliente — para clientes se cadastrarem, agendarem e acompanharem atendimentos
Arquitetura Multi-Tenant: cada empresa possui seu próprio tenant_id. O isolamento é feito via session('tenant_id') ?? 1 em todas as queries de negócio.

4. Sistema de Permissões

Perfis

  • admin — acesso total ao painel administrativo
  • employee — funcionário (funcionalidade futura completa)
  • client — acesso exclusivo ao portal do cliente

Como acessar permissão do usuário (CORRETO)

// Em views e controllers — usar a relação belongsToMany correta:
$user->permissions->first()?->name   // retorna: 'admin', 'employee' ou 'client'

// ERRADO — relação que NÃO existe no model User:
$user->userPermission->permission->name  // ← gera erro null

7. Funcionalidades Implementadas

Gestão de Serviços

  • CRUD completo via Livewire (sem reload)
  • Vínculo de funcionários por serviço (tabela user_services)
  • Exclusão com confirmação inline Sim/Não

Disponibilidade Semanal

  • Grade de 7 dias com toggle on/off por dia
  • Horário início/fim configurável por dia
  • Recorrente — configurar uma vez, vale toda semana

Agenda Admin (Semanal)

  • Visão Seg–Dom com navegação por semana
  • Filtro por funcionário
  • Cancelamento com confirmação inline
  • Hoje em azul, dias passados em cinza

Portal do Cliente

  • Login com validação de permissão client
  • Cadastro cria usuário + tenant + permissão em transação
  • Agendar: seleciona data → funcionários disponíveis → horário → confirma
  • Meus Agendamentos: próximos e histórico, com cancelamento

Relatório de Agendamentos

  • Filtros reativos: período, funcionário, serviço
  • Cards: total, realizados, cancelados, taxa de cancelamento
  • Impressão via window.print() com CSS que oculta UI

8. Convenções e Padrões

Badges de cor — Portal vs Admin

O portal usa Bootstrap 5 via CDN sem as classes customizadas do Materio. Classes bg-label-* não funcionam no portal — sempre usar rgba() inline. 
/* Portal — usar rgba inline */
<span style="background:rgba(255,171,0,.15);color:#a07800;">Texto</span>

/* Admin (Materio) — pode usar bg-label-* */
<span class="badge bg-label-warning">Texto</span>

Confirmações destrutivas — nunca usar confirm()

// Padrão Sim/Não inline via Livewire
public ?int $confirmingDeleteId = null;

public function confirmDelete(int $id): void {
    $this->confirmingDeleteId = $id;
}
public function delete(int $id): void {
    Model::findOrFail($id)->delete();
    $this->confirmingDeleteId = null;
}

11. Comandos Úteis

Criar usuário cliente via Tinker

php artisan tinker --execute="
\$user = App\Models\User::create([
    'name' => 'Nome Cliente',
    'email' => 'email@exemplo.com',
    'password' => bcrypt('password'),
]);
\$permId = DB::table('permissions')->where('name','client')->value('id');
DB::table('user_permissions')->insert([
    'tenant_id' => 1,
    'user_id' => \$user->id,
    'code_permission' => \$permId,
    'created_at' => now(),
    'updated_at' => now(),
]);
echo 'Criado! ID: ' . \$user->id;
"
